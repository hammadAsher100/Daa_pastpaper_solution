Master Theorem — All Cases with Worked Examples 5 Marks CLO-4
From Spring 2023 Paper
Solve the following recurrences using Master's Theorem:
(i) T(n) = 4T(n/2) + n²    (ii) T(n) = 16T(n/4) + n    (iii) T(n) = 2T(n/2) + n/log n    (iv) T(n) = 4T(n/2) + n!
📐 The Master Theorem — Standard Form

For recurrences of the form: T(n) = aT(n/b) + f(n), where a ≥ 1, b > 1, and f(n) is an asymptotically positive function.

Let n^(log_b a) be the "critical exponent". CASE 1: If f(n) = O(n^(log_b(a) - ε)) for some ε > 0 → f(n) grows SLOWER than n^(log_b a) → T(n) = Θ(n^(log_b a)) [recursion dominates] CASE 2: If f(n) = Θ(n^(log_b a) · log^k n) for some k ≥ 0 → f(n) grows AT THE SAME RATE as n^(log_b a) → T(n) = Θ(n^(log_b a) · log n) [split the work] CASE 3: If f(n) = Ω(n^(log_b(a) + ε)) for some ε > 0 AND a·f(n/b) ≤ δ·f(n) for some δ < 1 → f(n) grows FASTER than n^(log_b a) → T(n) = Θ(f(n)) [f(n) dominates]
🔑 Quick Decision Rule: Compute log_b(a). Compare with exponent of f(n). Lower exponent → Case 1. Same → Case 2. Higher exponent → Case 3.
✏️ Solution (i): T(n) = 4T(n/2) + n²

Step 1 — Extract parameters: a = 4, b = 2, f(n) = n²

Step 2 — Compute critical exponent: log_b(a) = log₂(4) = 2

Step 3 — Compare: f(n) = n² = Θ(n²) = Θ(n^(log₂4)) → same exponent k=0

Step 4 — Apply Case 2: T(n) = Θ(n² log n)

✅ Final Answer
T(n) = Θ(n² log n) — Case 2 applies since f(n) = Θ(n^(log₂4))
✏️ Solution (ii): T(n) = 16T(n/4) + n

Step 1: a = 16, b = 4, f(n) = n

Step 2: log_b(a) = log₄(16) = 2 (since 4² = 16)

Step 3: f(n) = n = n¹. Critical = n². Since 1 < 2, f(n) = O(n^(2-1)) where ε = 1

Step 4 — Case 1: T(n) = Θ(n^log₄16) = Θ(n²)

✅ Final Answer
T(n) = Θ(n²) — Case 1 applies since f(n) = O(n^(log₄16 - ε)) = O(n^(2−1))
✏️ Solution (iii): T(n) = 2T(n/2) + n/log n

Step 1: a = 2, b = 2, f(n) = n/log n

Step 2: log_b(a) = log₂(2) = 1, so critical = n

Step 3: f(n) = n/log n. Is this Case 1 or 2?

n/log n vs n¹ — n/log n is SLIGHTLY slower than n (by a log factor), but NOT by a polynomial factor ε. So no ε > 0 exists where n/log n = O(n^(1-ε)).

Step 4: This is a SPECIAL CASE — Master Theorem does NOT apply here.

⚠️ Reconstructed Answer: T(n) = 2T(n/2) + n/log n falls into the gap between Case 1 and Case 2 where the Master Theorem cannot be directly applied. The actual complexity is T(n) = Θ(n log log n) (derived via Akra-Bazzi or careful substitution). In exams, state: "Master Theorem does not directly apply; the gap exists between cases."
✏️ Solution (iv): T(n) = 4T(n/2) + n!

Step 1: a = 4, b = 2, f(n) = n!

Step 2: log_b(a) = log₂(4) = 2, critical = n²

Step 3: f(n) = n! which grows MUCH faster than n² polynomially (n! = Ω(n^(2+ε)) for any fixed ε)

Step 4 — Case 3: Check regularity: 4·(n/2)! ≤ δ·n! for large n? Yes, since (n/2)! << n!/4 for large n.

✅ Final Answer
T(n) = Θ(n!) — Case 3 applies since n! = Ω(n^(2+ε))

Recurrence by Substitution — Code Analysis 6 Marks CLO-4
From Spring 2023 Paper — Part (b)
Compute the time complexities of the following code segments using induction/substitution method.
Code (i): sum(n)
int sum(int n) { if(n == 1) return n*n; // Base case: O(1) else { for(i = 1; i < n*n; i++); // Loop runs n² times → O(n²) return (n/2) + sum(n-1); // 1 recursive call with (n-1) } }

Step 1 — Write the recurrence:

T(1) = c (constant)

T(n) = n² + T(n-1)     [loop costs n², one recursive call on n-1]

Step 2 — Apply iterative substitution:

T(n) = n² + T(n-1) = n² + (n-1)² + T(n-2) = n² + (n-1)² + (n-2)² + T(n-3) = ... = Σ(i=1 to n) i² + T(1) = n(n+1)(2n+1)/6 + c

Step 3 — Simplify: Σi² = n(n+1)(2n+1)/6 = Θ(n³)

✅ Final Answer
T(n) = Θ(n³)
The loop at each level costs O(n²), but the CUMULATIVE cost across all n recursive levels is Σi² ≈ n³/3 = Θ(n³).
Code (ii): xpowy(x, n)
int xpowy(int x, int n) { if (n == 0) return 1; // Base case if (n == 1) return x; // Base case if ((n % 2) == 0) return xpowy(x*x, n/2); // Even: one recursive call, size n/2 else return xpowy(x*x, n/2) * x; // Odd: one recursive call, size n/2 }

Step 1 — Write the recurrence:

T(1) = b (constant)

T(n) = T(n/2) + c    [ONE recursive call on n/2, constant work per level]

Step 2 — Apply iterative substitution:

T(n) = T(n/2) + c = T(n/4) + c + c = T(n/8) + 3c = T(n/2^k) + k·c Set n/2^k = 1 → k = log₂n T(n) = T(1) + c·log₂n = b + c·log₂n

Step 3 — Simplify: T(n) = Θ(log n)

✅ Final Answer
T(n) = Θ(log n)
This is the "fast exponentiation" / "repeated squaring" algorithm. Each call halves n, giving logarithmic depth.

Backtracking — 4 Queens Problem with Full State Space Tree 5 Marks CLO-2
Recurring in Spring 2023 and 2025 Papers
Explain backtracking design and also solution for 4 Queens Problem with proper illustrations of state space tree.
1. Backtracking Design Paradigm

Backtracking is a systematic algorithm design technique that builds a solution incrementally, one piece at a time, and abandons (backtracks) a partial solution as soon as it determines that it cannot lead to a valid complete solution.

Key Characteristics:

  • State-Space Tree: Represents all possible configurations. Each node is a partial solution; leaves are complete solutions or dead ends.
  • Pruning: When a partial solution violates constraints, we cut off that branch (prune), avoiding redundant computation.
  • More efficient than Brute Force: Does not explore all 2ⁿ or n! possibilities, only valid paths.
  • Applications: N-Queens, Sudoku, Maze solving, Subset Sum, Hamiltonian Circuit

Generic Backtracking Template:

Backtrack(candidate): if isSolution(candidate): output(candidate) return for each extension e of candidate: if isValid(e): place(e) Backtrack(e) remove(e) // BACKTRACK
2. The 4-Queens Problem

Problem: Place 4 queens on a 4×4 chessboard such that no two queens attack each other (no two queens share the same row, column, or diagonal).

Constraint Check (for placing queen at row r, col c):

  • No queen in same column: col[i] ≠ c for all placed rows i
  • No queen in diagonal: |row_i - r| ≠ |col_i - c| for all placed rows i
  • (Rows are automatically different since we place one queen per row)

Dijkstra's Shortest Path Algorithm 5 Marks CLO-2
Spring 2023 — Q3 Part b & Projected 2025
Explain the Greedy design paradigm and find the shortest path from vertex 'a' to all the vertices using Dijkstra's Algorithm. Also draw the resultant tree and shortest path lengths.
1. Greedy Design Paradigm

A Greedy Algorithm makes the locally optimal choice at each step, hoping to find the globally optimal solution.

  • Greedy-Choice Property: A locally optimal (greedy) choice can lead to a globally optimal solution.
  • Optimal Substructure: The global optimal solution contains optimal solutions to its sub-problems.
  • Top-Down approach: Greedy choices are made before solving sub-problems.
  • Examples: Dijkstra's, Prim's, Kruskal's, Fractional Knapsack, Activity Selection
2. Dijkstra's Algorithm

Dijkstra's finds the single-source shortest path from a source vertex to all other vertices in a weighted graph with non-negative edges.

// PSEUDOCODE: Dijkstra's Shortest Path Algorithm Dijkstra(Graph G, source s): for each vertex v in G: dist[v] = ∞ // Initialize all distances to infinity visited[v] = false dist[s] = 0 // Distance to source is 0 while unvisited vertices remain: u = vertex with minimum dist[u] (not yet visited) visited[u] = true for each neighbor v of u: if dist[u] + weight(u,v) < dist[v]: dist[v] = dist[u] + weight(u,v) // RELAX return dist[]
3. Worked Example (Spring 2023 Graph)

Graph from the exam paper (Source = vertex a):

Vertices: a, b, c, d, e, f, g Edges (undirected with weights): a—b: 4 b—c: 2 c—d: 10 d—e: 6 a—g: 2 b—g: 1 g—c: 3 g—e: 4 g—f: 8 e—f: 1 a—f: 5 Source: a

Initial State: dist[a]=0, all others = ∞

Iteration 1: Select a (dist=0). Relax neighbors:

  • dist[b] = 0+4 = 4
  • dist[g] = 0+2 = 2
  • dist[f] = 0+5 = 5

Iteration 2: Select g (dist=2). Relax:

  • dist[b] = min(4, 2+1) = 3 ✓ UPDATE
  • dist[c] = min(∞, 2+3) = 5
  • dist[e] = min(∞, 2+4) = 6
  • dist[f] = min(5, 2+8) = 5 (no change)

Iteration 3: Select b (dist=3). Relax:

  • dist[c] = min(5, 3+2) = 5 (no change)

Iteration 4: Select c (dist=5). Relax:

  • dist[d] = min(∞, 5+10) = 15

Iteration 5: Select f (dist=5). Relax:

  • dist[e] = min(6, 5+1) = 6 (no change)

Iteration 6: Select e (dist=6). Relax:

  • dist[d] = min(15, 6+6) = 12 ✓ UPDATE

Iteration 7: Select d (dist=12). No further updates.

Dynamic Programming — LCS (Longest Common Subsequence) 5 Marks CLO-2
Spring 2023 — Q3 Part c & 2025 Paper
Explain Dynamic Programming and its characteristics. Also describe LCS and explain the solution for:
X = {A, A, B, A, C, D, B, D, C, A, B}    Y = {A, B, A, C, C, A, D, C, B, A, B}
1. Dynamic Programming

Dynamic Programming (DP) is an algorithm design paradigm that solves complex problems by breaking them into overlapping subproblems and storing results to avoid recomputation.

Two Key Properties Required:

  • Optimal Substructure: The optimal solution contains optimal solutions to subproblems.
  • Overlapping Subproblems: The same subproblems are solved multiple times (unlike Divide & Conquer).

Two Approaches:

  • Top-Down (Memoization): Recursive with caching of results.
  • Bottom-Up (Tabulation): Iterative, fills table from smallest subproblems up.

Steps in a DP Solution: (1) Define subproblem structure, (2) Establish recurrence, (3) Fill table, (4) Extract solution.

2. LCS Definition and Recurrence

Given two sequences X and Y, find the longest subsequence common to both (elements don't need to be contiguous, but must maintain order).

LCS Recurrence: c[i][j] = 0 if i=0 or j=0 (base case) c[i][j] = c[i-1][j-1] + 1 if X[i] = Y[j] c[i][j] = max(c[i-1][j], c[i][j-1]) if X[i] ≠ Y[j]

Matrix Chain Multiplication — Pseudocode 5 Marks CLO-3
2025 Paper — Q4
Write pseudocode for Matrix Chain Order using Dynamic Programming.
Problem Definition

Given a chain of n matrices A₁, A₂, ..., Aₙ, determine the optimal parenthesization that minimizes the total number of scalar multiplications.

Key insight: Matrix multiplication is associative — order of parenthesization changes computation cost but not result.

Cost of multiplying A(p×q) × B(q×r) = p × q × r scalar multiplications

DP Recurrence
m[i][j] = 0 if i = j m[i][j] = min over all i ≤ k < j of: { m[i][k] + m[k+1][j] + p[i-1] × p[k] × p[j] } where p[0..n] is the dimensions array (matrix Ai has dimensions p[i-1] × p[i])
Pseudocode
// MATRIX-CHAIN-ORDER(p) — p is array of dimensions, length n+1 MATRIX-CHAIN-ORDER(p): n = p.length - 1 let m[1..n][1..n] and s[1..n-1][2..n] be new tables for i = 1 to n: m[i][i] = 0 // single matrix costs 0 for L = 2 to n: // L = chain length for i = 1 to n-L+1: j = i + L - 1 m[i][j] = ∞ for k = i to j-1: // try each split point k q = m[i][k] + m[k+1][j] + p[i-1] × p[k] × p[j] if q < m[i][j]: m[i][j] = q s[i][j] = k // record optimal split return m and s // To print optimal parenthesization: PRINT-OPTIMAL-PARENS(s, i, j): if i = j: print "A" + i else: print "(" PRINT-OPTIMAL-PARENS(s, i, s[i][j]) PRINT-OPTIMAL-PARENS(s, s[i][j]+1, j) print ")"
✅ Complexity
Time: O(n³) — three nested loops over chain length, start index, split point.
Space: O(n²) — two n×n tables (m and s).

KMP (Knuth-Morris-Pratt) Algorithm 5 Marks CLO-2
2025 Paper — Q3
Explain the KMP Algorithm with pseudocode and example.
Concept

KMP is a pattern matching algorithm that avoids redundant comparisons by precomputing a failure function (prefix table) for the pattern, which tells how far to slide the pattern when a mismatch occurs.

  • Time Complexity: O(n + m) where n = text length, m = pattern length
  • Advantage over Brute Force: Never re-examines already-matched text characters
  • Uses failure links (also called: failure function, π array, partial match table)
Step 1 — Build the Failure Function
// COMPUTE-PREFIX-FUNCTION(P) COMPUTE-PREFIX-FUNCTION(P): m = P.length π[1] = 0 k = 0 for q = 2 to m: while k > 0 AND P[k+1] ≠ P[q]: k = π[k] if P[k+1] = P[q]: k = k + 1 π[q] = k return π
Step 2 — KMP Matching
// KMP-MATCHER(T, P) KMP-MATCHER(T, P): n = T.length m = P.length π = COMPUTE-PREFIX-FUNCTION(P) q = 0 // number of characters matched for i = 1 to n: // scan text T while q > 0 AND P[q+1] ≠ T[i]: q = π[q] // follow failure link if P[q+1] = T[i]: q = q + 1 // next character matches if q = m: // full pattern found print "Pattern found at shift" (i - m) q = π[q] // prepare for next match
Example: Pattern = "AABAAB", Text = "AABAACAABAAB"

Failure Function for "AABAAB":

Pattern: A A B A A B Index: 1 2 3 4 5 6 π[i]: 0 1 0 1 2 3

π[2]=1 (longest proper prefix of "AA" that is also suffix = "A")
π[5]=2 (longest proper prefix of "AABAA" that is also suffix = "AA")
π[6]=3 (longest proper prefix of "AABAAB" that is also suffix = "AAB")

KMP matching skips redundant comparisons using π, finding match at position 8.

Fractional Knapsack (Greedy) 5 Marks CLO-2
2025 Paper — Q3 & Recurring
Explain Greedy design paradigm. Solve the Fractional Knapsack problem.
Algorithm
  • Step 1: Compute value/weight ratio (profit per unit weight) for each item.
  • Step 2: Sort items in non-increasing order of value/weight ratio.
  • Step 3: Greedily fill the knapsack — take as much of each item as possible.
  • Key difference from 0/1 Knapsack: Items CAN be broken (fractional amounts allowed)

Prim's Algorithm — Pseudocode & Example 5 Marks CLO-3
2025 Paper — Q4 & Spring 2021
Write pseudocode for Prim's algorithm. Explain with an example.
Concept

Prim's algorithm builds the Minimum Spanning Tree (MST) by starting from a single vertex and greedily adding the minimum weight edge that connects a new vertex to the growing MST.

  • Category: Greedy Algorithm
  • Resembles Dijkstra's Algorithm in structure
  • Works on connected weighted undirected graphs
  • Time Complexity: O(V²) with adjacency matrix, O(E log V) with min-heap
Pseudocode
// PRIM(G, w, r) — G: graph, w: weight function, r: root vertex PRIM(G, w, r): for each vertex u in G.V: key[u] = ∞ // minimum weight edge to connect u to MST parent[u] = NIL inMST[u] = false key[r] = 0 // start from root Q = min-priority queue of all vertices keyed by key[] while Q is not empty: u = EXTRACT-MIN(Q) // vertex with smallest key inMST[u] = true for each neighbor v of u: if inMST[v] = false AND w(u,v) < key[v]: parent[v] = u key[v] = w(u,v) // UPDATE key and reorder queue return parent[] // parent[] defines the MST edges
Example (Spring 2021 Graph)

Graph has vertices 0–3 with edges. Starting from vertex 0:

Initial: key = [0, ∞, ∞, ∞], MST = {}

Iteration 1: Extract vertex 0 (key=0). Update neighbors: set key[1]=5, key[2]=7 (example weights).

Iteration 2: Extract vertex 1 (min key=5). Add edge 0-1. Update new neighbors.

Continue until all vertices are in MST.

Result: MST edges form a tree with n-1 edges connecting all vertices at minimum total cost.

✅ Prim's vs Kruskal's
Prim's: Starts from a vertex, grows MST one vertex at a time. Better for dense graphs.
Kruskal's: Sorts all edges globally, adds edges without cycles. Better for sparse graphs. Can handle disconnected graphs (gives spanning forest).

Kruskal's Algorithm 3 Marks CLO-2
Algorithm Steps
  • Step 1: Sort all edges by weight in non-decreasing order.
  • Step 2: Initialize n disjoint sets (one per vertex).
  • Step 3: For each edge (u,v) in sorted order: if u and v are in different components (FIND returns different roots), add edge to MST and UNION the two components.
  • Step 4: Stop when MST has n-1 edges.
KRUSKAL(G): A = empty set // MST edge set for each vertex v: MAKE-SET(v) Sort edges by weight (non-decreasing) for each edge (u,v) in sorted order: if FIND-SET(u) ≠ FIND-SET(v): A = A ∪ {(u,v)} UNION(u, v) return A
✅ Key Facts
Time: O(E log E). Uses Union-Find data structure. Is a Greedy algorithm. Can run on disconnected graphs (produces spanning forest). Selects edges in increasing order of weight.

Key Definitions — Spring 2021 Style (5 × 1 mark) 5 Marks CLO-1
Divide and Conquer Strategy
Definition
Divide and Conquer is an algorithm design paradigm that works by dividing the input problem into two or more smaller disjoint subproblems, conquering each subproblem recursively (with base case for small inputs), and combining the solutions to solve the original problem. Examples: Merge Sort (O(n log n)), Quick Sort, Binary Search, Towers of Hanoi.
In-order Traversal
Definition
In-order traversal is a depth-first tree traversal technique that visits nodes in the order: Left subtree → Root → Right subtree. For a Binary Search Tree, in-order traversal produces nodes in sorted (non-decreasing) order.
Brute Force Approach
Definition
Brute Force is a straightforward algorithm design technique that tries all possible solutions exhaustively until a satisfactory solution is found. It is simple to implement and widely applicable but rarely yields efficient algorithms. Time complexity is typically exponential (Θ(2ⁿ) or Θ(n!)). Examples: Sequential Search (O(n)), exhaustive knapsack, TSP.
Minimum Spanning Tree (MST)
Definition
A Minimum Spanning Tree of a connected weighted undirected graph G = (V, E) is a spanning tree (a tree that includes all vertices) with the minimum total edge weight. An MST has exactly n-1 edges for n vertices, contains no cycles, and the graph remains connected. Algorithms: Prim's (O(E log V)) and Kruskal's (O(E log E)).
Bellman-Ford Algorithm
Definition
Bellman-Ford is a single-source shortest path algorithm that can handle graphs with negative edge weights (unlike Dijkstra's). It works by relaxing all edges V-1 times. Time complexity: O(V × E). It also detects negative weight cycles (if a path can be further relaxed after V-1 iterations, a negative cycle exists). It is NOT a greedy algorithm.

Branch & Bound — Travelling Salesman Problem 5 Marks CLO-2
Spring 2023 — Q3 Part d
Explain the characteristics of branch & bound algorithm design. Also find the best path for the Travelling Salesman Problem using branch & bound (given graph with 4 cities).
Branch & Bound — Characteristics

Branch & Bound (B&B) is an algorithm design paradigm used to solve optimization and combinatorial problems. It works by systematically exploring the search space while pruning subproblems that cannot lead to an optimal solution.

  • Branch: Divide the problem into smaller subproblems (branching).
  • Bound: Compute a bound (upper/lower) on the optimal solution for each subproblem. If a bound is worse than the current best known solution, prune that branch.
  • Best-First Search: Explore the most promising node first (minimum lower bound).
  • Completeness: Guaranteed to find the optimal solution (unlike greedy).
  • Applications: TSP, 0/1 Knapsack, Job Assignment, Integer Programming.
  • Difference from Backtracking: B&B uses bounding functions for optimization; Backtracking uses feasibility checks for constraint satisfaction.
TSP Worked Example (Spring 2023 Graph — 4 cities)

Cities: 1, 2, 3, 4. Given the adjacency matrix (from the exam diagram):

Cost Matrix: 1 2 3 4 1 [ ∞ 10 5 20 ] 2 [ 10 ∞ 20 6 ] 3 [ 5 20 ∞ 15 ] 4 [ 20 6 15 ∞ ] Goal: Find minimum-cost Hamiltonian circuit starting from city 1.

Step 1 — Compute Lower Bound (Reduced Matrix):

For each row: subtract the minimum element. Row mins: 5, 6, 5, 6. Total reduction = 22.

For each col of reduced matrix: subtract column minimums if > 0. Calculate total reduction.

Step 2 — Branch from Node 1:

Try edges 1→2, 1→3, 1→4. Compute new lower bounds for each branch.

Step 3 — Select minimum bound node and continue.

Result (exam graph): Optimal Tour = 1 → 3 → 2 → 4 → 1 with cost = 5+20+6+20 = 51, or find via actual matrix. Always verify by checking all tours systematically.

✅ Key Exam Points
B&B guarantees optimal solution. Lower bound = reduced matrix sum. Branch on unvisited cities. Prune if bound ≥ current best. Time: O(n!) worst case but much better in practice. Different from backtracking which only checks feasibility.

Dynamic Programming — Rod Cutting & Binomial Coefficient Pseudocode 5 Marks CLO-3
Spring 2023 — Q4 & Spring 2021 — Q5
Design pseudocode for binomial coefficient using Dynamic Programming technique. C(n,k) = n! / (n-k)! k!
Binomial Coefficient via DP

Pascal's Triangle recurrence: C(n,k) = C(n-1,k-1) + C(n-1,k) with base cases C(n,0) = C(n,n) = 1.

// BINOMIAL-COEFFICIENT(n, k) using DP — Bottom-Up Tabulation BINOMIAL_DP(n, k): let C[0..n][0..k] be a 2D table for i = 0 to n: for j = 0 to min(i, k): if j = 0 OR j = i: C[i][j] = 1 // Base case: C(n,0)=1 and C(n,n)=1 else: C[i][j] = C[i-1][j-1] + C[i-1][j] // Pascal's recurrence return C[n][k] // Time: O(n*k) | Space: O(n*k) // Example: C(5,2) = C(4,1) + C(4,2) = 4 + 6 = 10 ✓
Rod Cutting Problem (2021 Exam Pseudocode)

Given a rod of length n and prices p[1..n], find the maximum revenue by cutting the rod.

// DP-CUT-ROD(p, n) — Bottom-Up Dynamic Programming DP-CUT-ROD(p, n): let r[0..n] be a new array r[0] = 0 for j = 1 to n: q = -∞ for i = 1 to j: q = max(q, p[i] + r[j-i]) // best cut at position i r[j] = q // optimal revenue for length j return r[n] // Time: O(n²) | Space: O(n)
✅ Analysis of Exam Pseudocode
The 2021 exam asks to analyze the runtime of DP-CUT-ROD: The outer loop runs n times (j=1 to n), the inner loop runs j times. Total = 1+2+...+n = n(n+1)/2 = O(n²).

Rabin-Karp Algorithm — String Matching 5 Marks CLO-2
Spring 2021 — Q3 Part c
Explain the working mechanism of Rabin-Karp Algorithm using: Pattern = "leg", Text = "college". Values: a=1, b=2, ..., z=26.
Concept

Rabin-Karp uses hashing to find a pattern in a text. It computes a hash of the pattern and compares it with the hash of each text window. Only when hashes match does it verify character by character.

  • Time Complexity: O(n+m) average, O(nm) worst case (hash collisions)
  • Key Feature: Uses rolling hash — each new window hash computed in O(1) from previous
Worked Example
Pattern P = "leg" (length m=3) Text T = "college" (length n=7) Values: c=3, o=15, l=12, e=5, g=7 Simple hash = sum of character values: Pattern hash("leg") = 12 + 5 + 7 = 24 Window 1: "col" = 3+15+12 = 30 ≠ 24 → skip Window 2: "oll" = 15+12+12 = 39 ≠ 24 → skip Window 3: "lle" = 12+12+5 = 29 ≠ 24 → skip Window 4: "leg" = 12+5+7 = 24 = 24 → MATCH! Verify char by char: l=l✓ e=e✓ g=g✓ Pattern "leg" found at position 4 in "college".
✅ Key Points
Rabin-Karp uses a sliding window hash. Rolling hash avoids recomputing from scratch: new_hash = (old_hash - value[outgoing_char]) + value[incoming_char]. Hash match = possible match (needs verification). Hash mismatch = definite no match. Better than brute-force for multiple pattern search.
